# G01_F02_SF03: Request Password Reset

✅ MVP | **Token Management** | **Password Recovery**

## 📋 Description

User initiates password reset by entering email. System generates secure reset token and sends reset link via email.

## 🎯 API Specification

**Endpoint**: `POST /api/v1/auth/request-password-reset`

**Input**
```json
{
  "email": "user@example.com"
}
```

**Output (200 OK)**
```json
{
  "success": true,
  "data": {
    "message": "Password reset link sent to user@example.com"
  }
}
```

**Errors**: 404 (user not found), 429 (rate limit), 500 (error)

## 🗏️ Business Logic

1. **Validate Email**
   - Check email format
   - Return 400 if invalid

2. **Find User**
   - Query by email (case-insensitive)
   - Return 404 if not found (don't reveal if email registered)

3. **Apply Rate Limit**
   - Track per email: `password_reset_attempts:{email}`
   - Limit: 3 resets per hour
   - Return 429 if exceeded

4. **Invalidate Old Tokens**
   - Delete expired password reset tokens for user

5. **Generate Reset Token**
   - JWT with user_id, email, token_type="password_reset"
   - Expiry: 1 hour
   - Unique token per request

6. **Store Token**
   - Insert into password_reset_tokens table
   - Mark as not used

7. **Send Reset Email**
   - Queue email task
   - Include reset link with token
   - Link: `{FRONTEND}/reset-password?token={token}`

8. **Return Success**
   - HTTP 200 OK (even if user not found - security)
   - Generic message

## 🗄️ Database Schema

```sql
CREATE TABLE password_reset_tokens (
  id SERIAL PRIMARY KEY,
  user_id INT NOT NULL REFERENCES users(id),
  token VARCHAR(500) NOT NULL UNIQUE,
  expires_at TIMESTAMP NOT NULL,
  used_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 💻 Backend Implementation

```python
@router.post("/request-password-reset")
async def request_password_reset(
    data: dict,
    db: Session = Depends(get_db)
):
    """Request password reset"""
    try:
        result = await AuthService.request_password_reset(data.get("email"), db)
        return {"success": True, "data": result}
    except HTTPException:
        raise
    except Exception as e:
        logger.error(f"Password reset request error: {str(e)}")
        raise HTTPException(status_code=500, detail="Request failed")

async def request_password_reset(email: str, db: Session) -> dict:
    """Generate password reset token and send email"""
    try:
        # Check rate limit
        rate_key = f"password_reset_attempts:{email.lower()}"
        attempts = await redis_client.get(rate_key)
        if attempts and int(attempts) >= 3:
            raise HTTPException(status_code=429, detail="Too many reset attempts")
        
        # Find user (don't reveal if not found)
        user = db.query(User).filter(
            func.lower(User.email) == func.lower(email)
        ).first()
        
        if user:
            # Generate reset token
            reset_token = generate_jwt_token(
                data={"user_id": user.id, "email": user.email, "token_type": "password_reset"},
                expires_delta=timedelta(hours=1)
            )
            
            # Store token
            token_record = PasswordResetToken(
                user_id=user.id,
                token=reset_token,
                expires_at=datetime.utcnow() + timedelta(hours=1)
            )
            db.add(token_record)
            db.commit()
            
            # Send email
            await send_password_reset_email(user.email, reset_token)
            
            logger.info(f"Password reset requested for: {user.email}")
        
        # Increment counter regardless
        await redis_client.incr(rate_key)
        await redis_client.expire(rate_key, 3600)
        
        # Always return same message (security)
        return {"message": "If email exists, password reset link has been sent"}
    
    except HTTPException:
        raise
    except Exception as e:
        logger.error(f"Password reset error: {str(e)}")
        raise HTTPException(status_code=500, detail="Request failed")
```

## 📊’ Security

1. **Rate Limiting**: 3 resets per hour per email
2. **Token Expiry**: 1 hour (short-lived)
3. **Email Enumeration**: Generic response (don't reveal if email registered)
4. **One-time Use**: Mark token as used after reset
5. **Secure Link**: Include full reset URL in email with token
