## 📋 Description

Retrieve comprehensive metadata for a specific quiz, including title, description, question count, difficulty level, time limits, passing score, and user-specific progress information.

## 🗏️ Business Logic (10 Steps)

1. **Validate User Authentication**
   - Extract JWT token from Authorization header
   - Verify token validity and extract user_id
   - Return 401 Unauthorized if invalid

2. **Validate Quiz ID Parameter**
   - Check quiz_id is positive integer
   - Validate quiz_id format (numeric)
   - Return 400 Bad Request if invalid format

3. **Fetch Quiz Metadata**
   - Query: SELECT * FROM quizzes WHERE id = :quiz_id AND is_deleted = false
   - Return 404 Not Found if quiz not found or deleted

4. **Check Quiz Publication Status**
   - If quiz.is_published = false:
     - Check if user is admin or quiz creator
     - If neither, return 403 Forbidden
   - Non-published quizzes only visible to admins/creators

5. **Fetch Quiz Creator Information**
   - Query: SELECT id, username, avatar_url FROM users WHERE id = :created_by_id
   - Include creator details in response

6. **Count Quiz Questions**
   - Query: SELECT COUNT(*) FROM quiz_questions WHERE quiz_id = :quiz_id
   - Store in question_count

7. **Fetch User Progress & Attempts**
   - Query attempts: SELECT * FROM quiz_attempts WHERE user_id = :user_id AND quiz_id = :quiz_id
   - Calculate:
     - total_attempts = COUNT(*)
     - best_score = MAX(score)
     - average_score = AVG(score)
     - is_passed = MAX(is_passed) = true
     - last_attempt_at = MAX(started_at)

8. **Check Prerequisites**
   - Query: SELECT * FROM quiz_prerequisites WHERE quiz_id = :quiz_id
   - For each prerequisite quiz:
     - Check if user completed it (quiz_attempts.is_passed = true)
     - Set is_locked = true if any prerequisite incomplete
   - Get user.current_level from users table

9. **Determine User Permissions**
   - can_retake: Check quiz rules (unlimited by default)
   - is_locked: Check if user meets prerequisites and level requirements
   - Validate: user.current_level â‰¥ quiz.minimum_level_requirement

10. **Return Enriched Response**
    - Map all data to response schema
    - Include user_progress only if user has history
    - Cache for 10 minutes (key: quiz:{quiz_id}:user:{user_id})

### Service Layer

```python
# app/services/quiz_service.py
from sqlalchemy import and_
from sqlalchemy.orm import Session
from typing import Optional, Dict
from datetime import datetime
import logging

logger = logging.getLogger(__name__)

class QuizService:
    
    @staticmethod
    async def get_quiz_details(
        quiz_id: int,
        user_id: int,
        db: Session
    ) -> Dict:
        """
        Retrieve comprehensive quiz details for authenticated user.
        
        Args:
            quiz_id: ID of quiz to fetch
            user_id: ID of authenticated user
            db: Database session
            
        Returns:
            Dictionary with quiz details and user progress
            
        Raises:
            ValueError: If quiz not found
            PermissionError: If user unauthorized
        """
        # Fetch quiz
        quiz = db.query(Quiz).filter(
            and_(
                Quiz.id == quiz_id,
                Quiz.is_deleted == False
            )
        ).first()
        
        if not quiz:
            raise ValueError(f"Quiz {quiz_id} not found")
        
        # Check if user can view
        if not quiz.is_published:
            user = db.query(User).filter(User.id == user_id).first()
            if user_id != quiz.created_by_id and user.role != "admin":
                raise PermissionError("Not authorized to view this quiz")
        
        # Fetch creator info
        creator = db.query(User).filter(User.id == quiz.created_by_id).first()
        
        # Count questions
        question_count = db.query(func.count(QuizQuestion.id)).filter(
            QuizQuestion.quiz_id == quiz_id
        ).scalar() or 0
        
        # Fetch user attempts
        attempts = db.query(QuizAttempt).filter(
            and_(
                QuizAttempt.user_id == user_id,
                QuizAttempt.quiz_id == quiz_id
            )
        ).all()
        
        user_progress = None
        if attempts:
            scores = [a.score for a in attempts if a.score is not None]
            passed_attempts = [a for a in attempts if a.is_passed]
            
            user_progress = UserProgress(
                total_attempts=len(attempts),
                best_score=max(scores) if scores else None,
                best_score_at=passed_attempts[0].completed_at if passed_attempts else None,
                is_passed=len(passed_attempts) > 0,
                is_locked=False,
                can_retake=True,
                last_attempt_at=max([a.started_at for a in attempts]),
                average_score=sum(scores) / len(scores) if scores else None
            )
        
        # Check prerequisites
        prerequisites_raw = db.query(QuizPrerequisite).filter(
            QuizPrerequisite.quiz_id == quiz_id
        ).all()
        
        required_quizzes = []
        is_locked = False
        
        for prereq in prerequisites_raw:
            prereq_quiz = db.query(Quiz).filter(
                Quiz.id == prereq.prerequisite_quiz_id
            ).first()
            
            is_completed = db.query(QuizAttempt).filter(
                and_(
                    QuizAttempt.user_id == user_id,
                    QuizAttempt.quiz_id == prereq.prerequisite_quiz_id,
                    QuizAttempt.is_passed == True
                )
            ).first() is not None
            
            if not is_completed:
                is_locked = True
            
            required_quizzes.append(
                PrerequisiteQuiz(
                    quiz_id=prereq_quiz.id,
                    title=prereq_quiz.title,
                    is_completed=is_completed
                )
            )
        
        # Get user current level
        user = db.query(User).filter(User.id == user_id).first()
        user_current_level = user.current_level or 1
        
        # Check level requirement
        if user_current_level < quiz.minimum_level_requirement:
            is_locked = True
        
        prerequisites = Prerequisites(
            required_quizzes=required_quizzes,
            minimum_level=quiz.minimum_level_requirement,
            user_current_level=user_current_level
        )
        
        # Build response
        return {
            "quiz_id": quiz.id,
            "title": quiz.title,
            "description": quiz.description,
            "category": quiz.category,
            "difficulty": quiz.difficulty,
            "question_count": question_count,
            "time_limit_minutes": quiz.time_limit_minutes,
            "passing_score": quiz.passing_score,
            "points_reward": quiz.points_reward,
            "difficulty_level_range": quiz.difficulty_level_range,
            "created_by": UserBasicInfo(
                user_id=creator.id,
                username=creator.username,
                avatar_url=creator.avatar_url
            ),
            "created_at": quiz.created_at,
            "updated_at": quiz.updated_at,
            "user_progress": user_progress,
            "prerequisites": prerequisites,
            "tags": (quiz.tags or "").split(","),
            "thumbnail_url": quiz.thumbnail_url
        }
```

### API Handler

```python
# app/api/v1/quizzes.py
from fastapi import APIRouter, Depends, HTTPException, status, Path
import logging

router = APIRouter(prefix="/quizzes", tags=["quizzes"])
logger = logging.getLogger(__name__)

@router.get("/{quiz_id}", response_model=dict)
async def get_quiz_details(
    quiz_id: int = Path(..., ge=1, description="Quiz ID"),
    user: User = Depends(get_current_user),
    db: Session = Depends(get_db)
) -> dict:
    """
    Get comprehensive details for a specific quiz.
    
    Path Parameters:
    - quiz_id: ID of quiz to fetch (required, must be > 0)
    
    Returns:
    - Detailed quiz metadata with user progress
    - Includes prerequisites and eligibility status
    
    Responses:
    - 200: Quiz details retrieved successfully
    - 404: Quiz not found
    - 401: Unauthorized
    - 403: User not allowed to view quiz
    """
    try:
        logger.info(f"User {user.id} fetching quiz {quiz_id} details")
        
        quiz_data = await QuizService.get_quiz_details(
            quiz_id=quiz_id,
            user_id=user.id,
            db=db
        )
        
        return {
            "success": True,
            "data": quiz_data
        }
    
    except ValueError as e:
        logger.warning(f"Quiz not found: {str(e)}")
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail=str(e)
        )
    
    except PermissionError as e:
        logger.warning(f"Permission denied: {str(e)}")
        raise HTTPException(
            status_code=status.HTTP_403_FORBIDDEN,
            detail=str(e)
        )
    
    except Exception as e:
        logger.error(f"Failed to fetch quiz details: {str(e)}", exc_info=True)
        raise HTTPException(
            status_code=status.HTTP_500_INTERNAL_SERVER_ERROR,
            detail="Failed to fetch quiz details"
        )
```

## 📊’ Security Considerations

1. **Authentication Required**
   - Valid JWT token required in Authorization header
   - Extract user_id from claims
   - Return 401 if token invalid/expired

2. **Authorization Checks**
   - Non-published quizzes only visible to:
     - Quiz creator (created_by_id == user_id)
     - Admin users (user.role == "admin")
   - Return 403 Forbidden if user unauthorized
   - Prevent access to deleted quizzes

3. **Data Visibility**
   - User progress shown only for current user
   - Creator information sanitized (no passwords/emails)
   - Prerequisites must be validated on backend
   - Never expose correct answers before completion

4. **Input Validation**
   - quiz_id must be positive integer
   - Validate path parameter format
   - Return 400 if invalid format
   - Use parameterized queries to prevent SQL injection

5. **Rate Limiting**
   - Implement per-user rate limit (100 requests/min)
   - Cache individual quiz details (10 min TTL)
   - Return 429 Too Many Requests if exceeded
