# G02_F04_SF08: Reset User Scores

✅ MVP | **Admin** | **Testing Only** | **Difficulty**: Easy

## 📋 Description
Admin-only endpoint to reset user's quiz scores for testing purposes. Soft delete approach (archive old data).

## 🎯 API Specification

**Endpoint**: `POST /api/v1/users/{user_id}/reset-scores`

**Input**: `{ "confirm": true }`

**Output (200 OK)**:
```json
{
  "success": true,
  "message": "Scores reset successfully",
  "data": {
    "attempts_archived": 45,
    "reset_at": "2026-05-04T10:30:00Z"
  }
}
```

## 🗏️ Business Logic (5 Steps)

1. Validate JWT + admin role
2. Require `confirm=true` to prevent accidents
3. Archive quiz_attempts to `archived_quiz_attempts` table
4. Delete from active `quiz_attempts` table (or mark is_archived=true)
5. Log action with admin_id, user_id, count
