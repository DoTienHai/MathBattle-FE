# G05_F01_SF04: Redeem Points

✅ MVP | **User Points** | **Difficulty**: Medium

## 📋 Description
User redeems points for rewards. Checks balance, deducts points, marks reward as redeemed.

## 🎯 API Specification

**Endpoint**: `POST /api/v1/users/{user_id}/points/redeem`

**Input**:
```json
{
  "reward_id": 12,
  "points_to_spend": 500
}
```

**Output (200)**:
```json
{
  "success": true,
  "data": {
    "transaction_id": 791,
    "reward_name": "Premium Pass (1 Month)",
    "points_spent": 500,
    "new_balance": 1950,
    "reward_applied_at": "2026-05-04T10:30:00Z"
  }
}
```

## 🗏️ Business Logic (7 Steps)

1. Validate JWT + authorization (own user)
2. Fetch reward, return 404 if not found
3. Verify user balance â‰¥ points_to_spend
4. Check reward availability
5. INSERT transaction: user_id, -points, "reward_redemption"
6. UPDATE user_points: balance -= points
7. Apply reward (e.g., activate premium, unlock content)
