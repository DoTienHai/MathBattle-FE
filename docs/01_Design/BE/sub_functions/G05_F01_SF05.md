# G05_F01_SF05: Get Point History

✅ MVP | **User Points** | **Transactions** | **Difficulty**: Medium

## 📋 Description
Paginated transaction history: points earned, spent, reason, timestamp. Useful for auditing and user understanding.

## 🎯 API Specification

**Endpoint**: `GET /api/v1/users/{user_id}/points/history`

**Query**: `page`, `page_size`, `reason` (filter), `sort_by` (date, amount)

**Output (200)**:
```json
{
  "success": true,
  "data": {
    "transactions": [
      {
        "transaction_id": 789,
        "points": 100,
        "reason": "quiz_completion",
        "reference_id": 456,
        "description": "Quiz 'Basic Addition' completed",
        "balance_after": 2550,
        "timestamp": "2026-05-04T10:30:00Z"
      }
    ],
    "pagination": {"page": 1, "total": 245}
  }
}
```

## 🗏️ Business Logic (7 Steps)

1. Validate JWT + authorization
2. Build query: `SELECT * FROM point_transactions WHERE user_id = :user_id`
3. Apply optional reason filter
4. Apply sorting: date DESC or amount (ASC/DESC)
5. Paginate: offset, limit
6. Calculate balance_after for each transaction
7. Return paginated history
