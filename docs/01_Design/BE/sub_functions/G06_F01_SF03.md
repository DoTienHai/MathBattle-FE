# G06_F01_SF03: Send Friend Request

✅ MVP | **Social** | **Difficulty**: Medium

## 📋 Description
Send friend request to another user with optional message. Prevents duplicate requests.

## 🎯 API Specification

**Endpoint**: `POST /api/v1/users/{recipient_id}/friend-requests`

**Input**:
```json
{
  "message": "Hi! Let's practice together!"
}
```

**Output (201)**:
```json
{
  "success": true,
  "data": {
    "request_id": 567,
    "recipient_id": 124,
    "status": "pending",
    "sent_at": "2026-05-04T10:30:00Z"
  }
}
```

## 🗏️ Business Logic (8 Steps)

1. Validate JWT + extract sender_id
2. Fetch recipient user, return 404 if not found
3. Check not already friends: query friendships table
4. Check no pending request already exists
5. Check sender not blocked by recipient
6. INSERT friend_request: sender_id, recipient_id, message, status='pending'
7. Send notification to recipient
8. Return request_id
