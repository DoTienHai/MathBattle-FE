# G06_F01_SF04: Accept or Reject Friend Request

✅ MVP | **Social** | **Difficulty**: Easy

## 📋 Description
Accept or reject incoming friend request. Creates friendship record if accepted.

## 🎯 API Specification

**Endpoint**: `POST /api/v1/friend-requests/{request_id}/{action}`

**Path Parameters**: `action`: "accept" | "reject"

**Output (200)**:
```json
{
  "success": true,
  "message": "Friend request accepted",
  "data": {
    "request_id": 567,
    "status": "accepted",
    "processed_at": "2026-05-04T10:30:00Z"
  }
}
```

## 🗏️ Business Logic (6 Steps)

1. Validate JWT + authorization (recipient only)
2. Fetch friend_request, return 404 if not found
3. Update status: status = 'accepted' | 'rejected'
4. If accepted: INSERT into friendships table
5. Send notification to sender
6. Return updated status
