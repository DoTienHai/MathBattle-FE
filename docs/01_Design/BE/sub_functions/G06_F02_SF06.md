# G06_F02_SF06: Get Messages

✅ MVP | **Social** | **Messaging** | **Pagination** | **Difficulty**: Medium

## 📋 Description
Retrieve paginated conversation history with specific user. Marks messages as read.

## 🎯 API Specification

**Endpoint**: `GET /api/v1/messages/conversations/{user_id}`

**Query**: `page`, `page_size`, `sort_by` (newest_first, oldest_first)

**Output (200)**:
```json
{
  "success": true,
  "data": {
    "conversation": [
      {
        "message_id": 890,
        "sender_id": 123,
        "content": "Hey!",
        "sent_at": "2026-05-04T10:00:00Z",
        "read": true,
        "read_at": "2026-05-04T10:01:00Z"
      }
    ],
    "pagination": {"page": 1, "total": 45}
  }
}
```

## 🗏️ Business Logic (7 Steps)

1. Validate JWT + authorization (participants only)
2. Check friendship exists
3. Query messages WHERE (sender_id=user1 AND recipient_id=user2) OR (sender_id=user2 AND recipient_id=user1)
4. Order by sent_at DESC
5. Paginate: limit, offset
6. Mark unread messages as read
7. Return conversation with read status
