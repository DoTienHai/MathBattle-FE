## 📋 Description

Send encrypted direct messages between friends with optional attachments. Includes rate limiting, profanity filtering, delivery confirmation, and typing indicators. Messages are end-to-end encrypted.

---

## 🎯 Detailed Requirements

### Input Parameters

```python
# Request Body
{
  "recipient_id": integer (required, min: 1),
  "message_text": string (required, min: 1, max: 5000),
  "attachments": [
    {
      "file_id": string (optional, URL to uploaded file),
      "file_type": string (image|document|media)
    }
  ] (optional, max: 5 attachments)
}

# Example:
{
  "recipient_id": 456,
  "message_text": "That's great! Let me know your score on the math quiz.",
  "attachments": []
}
```

## 🗏️ Business Logic (12 Steps)

1. **Authenticate Sender**
   - Verify JWT token is valid
   - Extract sender_id from token claims
   - Verify account not banned
   - Return 401 if unauthorized

2. **Validate Request Parameters**
   - Check recipient_id is positive integer
   - Check message_text length âˆˆ [1, 5000] chars
   - Check attachments count â‰¤ 5
   - Validate each attachment has file_id and file_type
   - Return 422 if validation fails

3. **Verify Recipient Exists**
   - Query user by recipient_id
   - Confirm recipient account is not deleted/banned
   - Return 404 if recipient doesn't exist

4. **Check Friendship Status**
   - Query friendships table for sender_id â†’ recipient_id
   - Verify status = "accepted"
   - Return 403 if not friends

5. **Apply Profanity Filter**
   - Check message_text for offensive language
   - If profanity found: Replace with *** or flag message
   - Log potential violations

6. **Check Rate Limiting**
   - Query recent messages from sender in last minute
   - If > 10 messages in 60 seconds: Return 429 Too Many Requests
   - Per-user rate limit to prevent spam

7. **Generate Message ID**
   - Create unique message_id (prefix: "msg_" + UUID)
   - Generate encryption nonce (random 16 bytes)

8. **Encrypt Message Content**
   - Get recipient's public key from users table
   - Use AES-256-GCM encryption
   - Encrypt message_text with recipient's key
   - Store encrypted text in database

9. **Create Message Record**
   - INSERT into direct_messages table
   - Store: sender_id, recipient_id, encrypted_text, attachments, nonce, created_at
   - Set status = "sent", delivered_at = null

10. **Queue Delivery Notification**
    - Insert into message_notifications table
    - Queue push notification (async task)
    - Include in unread messages count

11. **Update Sender's Last Activity**
    - SET users.last_activity_at = NOW()
    - Update conversation preview

12. **Return Success Response**
    - Include message_id, status, timestamps
    - Return HTTP 201 Created

## 🗄️ Database Schema

### Tables Required

**users**
```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(100) UNIQUE NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  public_key TEXT,
  is_banned BOOLEAN DEFAULT false,
  last_activity_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**direct_messages**
```sql
CREATE TABLE direct_messages (
  id VARCHAR(50) PRIMARY KEY,
  sender_id INT NOT NULL REFERENCES users(id),
  recipient_id INT NOT NULL REFERENCES users(id),
  encrypted_text TEXT NOT NULL,
  encryption_nonce VARCHAR(32) NOT NULL,
  attachments JSONB DEFAULT '[]',
  status VARCHAR(20) DEFAULT 'sent' CHECK (status IN ('sent', 'delivered', 'read')),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  delivered_at TIMESTAMP,
  read_at TIMESTAMP,
  deleted_by_sender BOOLEAN DEFAULT false,
  deleted_by_recipient BOOLEAN DEFAULT false
);
```

**message_attachments**
```sql
CREATE TABLE message_attachments (
  id SERIAL PRIMARY KEY,
  message_id VARCHAR(50) NOT NULL REFERENCES direct_messages(id),
  file_url VARCHAR(500) NOT NULL,
  file_type VARCHAR(50) CHECK (file_type IN ('image', 'document', 'media')),
  file_size INT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**conversations**
```sql
CREATE TABLE conversations (
  id SERIAL PRIMARY KEY,
  user_id_1 INT NOT NULL REFERENCES users(id),
  user_id_2 INT NOT NULL REFERENCES users(id),
  last_message_id VARCHAR(50) REFERENCES direct_messages(id),
  last_message_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id_1, user_id_2),
  CHECK (user_id_1 < user_id_2)
);
```

**message_notifications**
```sql
CREATE TABLE message_notifications (
  id SERIAL PRIMARY KEY,
  message_id VARCHAR(50) NOT NULL REFERENCES direct_messages(id),
  recipient_id INT NOT NULL REFERENCES users(id),
  is_read BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Indexes
```sql
CREATE INDEX idx_direct_messages_sender_id ON direct_messages(sender_id);
CREATE INDEX idx_direct_messages_recipient_id ON direct_messages(recipient_id);
CREATE INDEX idx_direct_messages_created_at ON direct_messages(created_at DESC);
CREATE INDEX idx_direct_messages_status ON direct_messages(status);
CREATE INDEX idx_conversations_user_id_1 ON conversations(user_id_1);
CREATE INDEX idx_conversations_user_id_2 ON conversations(user_id_2);
CREATE INDEX idx_conversations_last_message_at ON conversations(last_message_at DESC);
CREATE INDEX idx_message_notifications_recipient_id ON message_notifications(recipient_id);
CREATE INDEX idx_message_notifications_is_read ON message_notifications(is_read);
```

## 📊’ Security Considerations

1. **End-to-End Encryption** - AES-256-GCM, never store plaintext
2. **Rate Limiting** - 10 messages per user per minute
3. **Profanity Filtering** - Filter offensive language, log violations
4. **Friendship Verification** - Only message friends
5. **Attachment Security** - Validate file types, scan for malware
6. **Privacy Controls** - Soft delete, 90-day expiry, no tracking without consent
