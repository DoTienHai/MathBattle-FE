# G06_F03_SF01: Get Notifications

✅ MVP | **Social** | **Notifications** | **Pagination** | **Difficulty**: Medium

## 📋 Description
Retrieve paginated user notifications: friend requests, achievements, challenge invites, system messages.

## 🎯 API Specification

**Endpoint**: `GET /api/v1/notifications`

**Query**: `page`, `page_size`, `type` (all|friend_request|achievement|challenge|system), `unread_only` (bool)

**Output (200)**:
```json
{
  "success": true,
  "data": {
    "notifications": [
      {
        "notification_id": 111,
        "type": "friend_request",
        "title": "Friend Request",
        "content": "jane_smith sent you a friend request",
        "related_id": 567,
        "read": false,
        "created_at": "2026-05-04T10:00:00Z"
      },
      {
        "notification_id": 112,
        "type": "achievement",
        "title": "Achievement Unlocked",
        "content": "You earned 'Quiz Master' achievement!",
        "read": false,
        "created_at": "2026-05-04T09:30:00Z"
      }
    ],
    "unread_count": 5,
    "pagination": {"page": 1, "total": 42}
  }
}
```

## 🗏️ Business Logic (8 Steps)

1. Validate JWT + extract user_id
2. Query notifications: WHERE user_id = :user_id
3. Apply type filter if provided
4. If unread_only=true: WHERE read=false
5. Order by created_at DESC
6. Paginate: limit, offset
7. Get unread count: COUNT WHERE read=false
8. Return notifications with metadata
