# G06_F03_SF02: Mark Notification as Read

✅ MVP | **Social** | **Notifications** | **Difficulty**: Easy

## 📋 Description
Mark single or multiple notifications as read. Updates read timestamp.

## 🎯 API Specification

**Endpoint**: `PUT /api/v1/notifications/{notification_id}/read`

Or bulk: `PUT /api/v1/notifications/bulk-read`

**Input (bulk)**:
```json
{
  "notification_ids": [111, 112, 113]
}
```

**Output (200)**:
```json
{
  "success": true,
  "message": "Notification(s) marked as read",
  "data": {
    "updated_count": 3,
    "unread_remaining": 2
  }
}
```

## 🗏️ Business Logic (4 Steps)

1. Validate JWT + extract user_id
2. Fetch notification(s), return 404 if not found
3. UPDATE notifications: read=true, read_at=NOW() WHERE notification_id IN (...)
4. Return updated count
