# G08_F01_SF03: Get Daily Quiz Performance

✅ MVP | **Analytics** | **Performance Tracking**

## 📋 Description

Retrieve daily quiz performance metrics for a specific date range. Show performance trends, best/worst performing categories, and time-based analytics.

## 🎯 API Specification

**Endpoint**: `GET /api/v1/analytics/daily-performance?start_date=2026-04-01&end_date=2026-05-04&user_id={user_id}`

**Query Parameters**
```
- start_date: date (YYYY-MM-DD, required)
- end_date: date (YYYY-MM-DD, required)
- user_id: integer (optional, defaults to current user)
- category: string (optional, filter by category)
```

**Output (200 OK)**
```json
{
  "success": true,
  "data": {
    "start_date": "2026-04-01",
    "end_date": "2026-05-04",
    "total_days": 34,
    "active_days": 22,
    "daily_metrics": [
      {
        "date": "2026-05-04",
        "quizzes_attempted": 3,
        "quizzes_passed": 2,
        "average_score": 85.5,
        "total_time_minutes": 45,
        "points_earned": 250
      }
    ],
    "best_performing_category": "arithmetic",
    "worst_performing_category": "geometry",
    "total_quizzes": 67,
    "total_points": 4500
  }
}
```

## 🗏️ Business Logic (9 Steps)

1. **Authenticate User** - Verify JWT
2. **Validate Date Range** - start_date < end_date, max 90 days
3. **Check User Access** - Own data or admin
4. **Query Quiz Attempts** - Filter by date range
5. **Group by Date** - Aggregate metrics per day
6. **Calculate Metrics** - attempts, passed, avg score, time, points per day
7. **Analyze Trends** - Best/worst categories
8. **Sort by Date** - Return in chronological order
9. **Return Response** - Compiled daily metrics

## 🗄️ Database Schema

Existing: `quiz_attempts`, `quiz_questions`

Query Pattern:
```sql
SELECT 
  DATE(quiz_attempts.completed_at) as date,
  COUNT(*) as attempts,
  SUM(CASE WHEN is_passed=true THEN 1 ELSE 0 END) as passed,
  AVG(score) as avg_score,
  SUM(time_spent_seconds)/60 as total_minutes,
  SUM(points_earned) as points
FROM quiz_attempts
WHERE user_id = :user_id 
  AND completed_at >= :start_date
  AND completed_at <= :end_date
GROUP BY DATE(quiz_attempts.completed_at)
ORDER BY date DESC
```

## 💻 Backend Implementation

```python
# app/api/v1/analytics.py
@router.get("/daily-performance")
async def get_daily_performance(
    start_date: str = Query(..., regex="^\d{4}-\d{2}-\d{2}$"),
    end_date: str = Query(..., regex="^\d{4}-\d{2}-\d{2}$"),
    user_id: Optional[int] = None,
    category: Optional[str] = None,
    current_user: User = Depends(get_current_user),
    db: Session = Depends(get_db)
):
    """Get daily quiz performance"""
    try:
        # Parse dates
        start = datetime.strptime(start_date, "%Y-%m-%d")
        end = datetime.strptime(end_date, "%Y-%m-%d")
        
        if start > end:
            raise HTTPException(status_code=400, detail="Start date must be before end date")
        
        if (end - start).days > 90:
            raise HTTPException(status_code=400, detail="Date range max 90 days")
        
        # Default to current user
        if not user_id:
            user_id = current_user.id
        elif user_id != current_user.id and current_user.role != "admin":
            raise HTTPException(status_code=403, detail="Unauthorized")
        
        # Query daily metrics
        query = db.query(
            func.date(QuizAttempt.completed_at).label("date"),
            func.count().label("attempts"),
            func.sum(case([(QuizAttempt.is_passed == True, 1)], else_=0)).label("passed"),
            func.avg(QuizAttempt.score).label("avg_score"),
            func.sum(QuizAttempt.time_spent_seconds) / 60.0,
            func.sum(QuizAttempt.points_earned).label("points")
        ).filter(
            and_(
                QuizAttempt.user_id == user_id,
                QuizAttempt.completed_at >= start,
                QuizAttempt.completed_at <= end
            )
        ).group_by(func.date(QuizAttempt.completed_at)).order_by(
            func.date(QuizAttempt.completed_at).desc()
        )
        
        daily_metrics = query.all()
        
        daily_data = []
        for metric in daily_metrics:
            daily_data.append({
                "date": metric.date.isoformat(),
                "quizzes_attempted": metric.attempts or 0,
                "quizzes_passed": metric.passed or 0,
                "average_score": round(float(metric.avg_score), 1) if metric.avg_score else 0,
                "total_time_minutes": int(metric[4]) if metric[4] else 0,
                "points_earned": int(metric.points) if metric.points else 0
            })
        
        active_days = len(daily_data)
        total_days = (end - start).days + 1
        
        return {
            "success": True,
            "data": {
                "start_date": start_date,
                "end_date": end_date,
                "total_days": total_days,
                "active_days": active_days,
                "daily_metrics": daily_data,
                "total_quizzes": sum(d["quizzes_attempted"] for d in daily_data),
                "total_points": sum(d["points_earned"] for d in daily_data)
            }
        }
    except HTTPException:
        raise
    except Exception as e:
        logger.error(f"Performance analytics error: {str(e)}")
        raise HTTPException(status_code=500, detail="Failed to fetch analytics")
```
